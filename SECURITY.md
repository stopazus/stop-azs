# Security Policy

## Supported Versions

The following versions of this project are currently supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| latest (main branch) | :white_check_mark: |

This project is actively maintained and security updates are applied to the main branch.

## Reporting a Vulnerability

We take the security of our project seriously. If you discover a security vulnerability, please report it responsibly.

### How to Report

**Preferred Method: GitHub Security Advisories**

1. Navigate to the [Security tab](https://github.com/stopazus/stop-azs/security) of this repository
2. Click "Report a vulnerability"
3. Fill out the private vulnerability report form
4. Submit your report

This method allows us to discuss and fix the issue privately before public disclosure.

**Alternative Methods:**

- Create a private security advisory through GitHub's interface
- Contact the repository maintainer [@stopazus](https://github.com/stopazus) directly through GitHub

### What to Include

When reporting a vulnerability, please include:

- A clear description of the vulnerability
- Steps to reproduce the issue
- Potential impact of the vulnerability
- Any suggested fixes or mitigations (optional)
- Your contact information for follow-up questions

### Response Time Expectations

- **Initial Response**: Within 48 hours of receiving your report
- **Status Update**: Within 7 days with an assessment of the vulnerability
- **Resolution Timeline**: Depends on severity
  - **Critical**: Within 7 days
  - **High**: Within 30 days
  - **Medium**: Within 60 days
  - **Low**: Within 90 days

### What to Expect

**If the vulnerability is accepted:**
- We will work on a fix and keep you informed of our progress
- We will credit you in the security advisory (unless you prefer to remain anonymous)
- We will coordinate disclosure timing with you
- We will release a security update and publish a security advisory

**If the vulnerability is declined:**
- We will explain our reasoning
- We may suggest alternative approaches or clarifications
- You are welcome to discuss our assessment

### Security Update Process

1. Security vulnerabilities are addressed through the GitHub Security Advisory workflow
2. Fixes are developed and tested in private
3. Security updates are released with a coordinated disclosure
4. A security advisory is published with details and credits

### Scope

This security policy applies to:
- All code in the main branch
- All released versions
- All dependencies managed by this project

### Additional Security Measures

This repository has the following security features enabled:
- **Dependabot**: Automated dependency updates for Python and GitHub Actions
- **CodeQL Analysis**: Automated code scanning for security vulnerabilities
- **Security-Extended Queries**: Enhanced security scanning with comprehensive query suite

For questions about this security policy, please open an issue or contact [@stopazus](https://github.com/stopazus).
